# 세션 스크립트

## 오프닝

오늘은 AI를 잘 쓰는 법이 아니라, AI와 함께 일을 끝내는 방식을 실습합니다. Claude Desktop을 이용해서 문제를 정의하고, PRD를 만들고, 설계를 문서화하고, 코드를 만들고, 리뷰까지 가보겠습니다.

## 문제 정의 세션 멘트

좋은 AI 실습 주제는 모델이 똑똑해 보여야 하는 주제가 아니라, 반복되고 기준이 있는 일을 다루는 주제입니다. 민원/FAQ 자동화는 그 조건에 잘 맞습니다.

## PRD 세션 멘트

PRD는 개발자만 보는 문서가 아닙니다. AI에게도 같은 기준을 읽히기 위한 핵심 문서입니다. 문서가 정리되면 구현 품질도 훨씬 안정적입니다.

## 구현 세션 멘트

한 번에 큰 걸 시키기보다, 작은 작업 단위로 나누는 것이 중요합니다. 오늘은 목록 API, 상세 API, 추천 생성 API, 그리고 UI 연결 순서로 갑니다.

## 품질 개선 세션 멘트

AI 결과가 마음에 안 든다면 보통 모델 성능보다 지시의 모호함이 원인인 경우가 많습니다. 오늘은 프롬프트를 어떻게 더 구체화하는지 같이 보겠습니다.
