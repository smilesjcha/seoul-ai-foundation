# 상세 커리큘럼

## 1. 환경 세팅 & OT

- Claude Desktop에서 프로젝트 폴더를 여는 방법
- GitHub 저장소 구조 설명
- 오늘 만들 결과물 미리 보기
- 실습 규칙 안내

강조 포인트:

- 수강생이 "어디서부터 손대야 할지 모르겠다"는 상태가 되지 않도록, 폴더 구조와 산출물 지도를 먼저 보여줍니다.
- Claude에게 막연한 질문보다 역할, 산출물, 제약 조건을 주는 방식이 중요함을 강조합니다.

## 2. 문제 정의 & 서비스 이해

- 민원/FAQ 자동화가 왜 좋은 교육 예제인지 설명
- 운영자, 시민, 관리자 관점에서 문제를 나눔
- 현재 프로세스의 병목을 텍스트로 정의

실습 목표:

- Claude와 함께 Problem Statement를 작성합니다.

## 3. PRD 작성

- 사용자 유형 정의
- 핵심 유저 플로우 작성
- MVP 범위 제한
- 성공 지표 작성

실습 목표:

- `docs/04-prd-faq-automation.md` 수준의 문서를 스스로 확장할 수 있도록 합니다.

## 4. 설계 문서화

- 아키텍처 개요
- API 흐름
- 데이터 모델
- 프롬프트 흐름

실습 목표:

- "Claude가 작성한 PRD를 Claude와 함께 설계 문서로 바꾸는 법"을 익힙니다.

## 5. 프로토타입 구현

- 백엔드 API와 Mock 데이터 연결
- 프론트엔드 목록/상세/응답 초안 화면 구현
- Claude에게 작은 작업 단위로 구현 요청하기

실습 목표:

- 기능 v1, v2까지 빠르게 완성합니다.

## 6. AI 품질 개선

- 분류 프롬프트 개선
- 응답 초안의 톤/안전성 개선
- Edge case 처리 기준 정리

## 7. 코드 리뷰 & 테스트

- Claude / Codex에 리뷰 요청하기
- 회귀가 생길 수 있는 지점 찾기
- 최소 테스트 케이스 설계

## 8. 확장 논의

- 실제 기관 도입 시 필요한 요소
- 운영자 승인 플로우
