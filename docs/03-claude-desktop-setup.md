# Claude Desktop 실습 환경 가이드

## 목표

수강생이 Claude Desktop을 단순 대화 도구가 아니라, 프로젝트 문맥을 이해하고 문서와 코드를 함께 다루는 워크스페이스 도구로 사용할 수 있게 만드는 것이 목표입니다.

## 준비물

- Claude Desktop
- Git
- Node.js 20+
- GitHub 계정
- 코드 편집기

## 권장 준비 순서

1. GitHub에서 저장소를 clone합니다.
2. Claude Desktop에서 프로젝트 폴더를 엽니다.
3. Claude에게 프로젝트 구조를 요약하게 합니다.
4. `docs/`와 `claude/` 폴더를 먼저 읽게 한 뒤 구현을 시작합니다.

## 강의 중 강조할 프롬프트 원칙

- 역할을 먼저 준다.
- 산출물을 명시한다.
- 범위를 제한한다.
- 기존 파일을 기준으로 작업하게 한다.
- 검토 기준까지 함께 준다.

## 예시 프롬프트

```md
이 저장소는 민원/FAQ 자동화 워크숍용 프로젝트야.

1. docs/04-prd-faq-automation.md를 기준으로
2. apps/api에서 먼저 MVP API를 구현하고
3. data/mock에 있는 데이터를 사용하고
4. 변경 파일과 이유를 간단히 설명해줘.
```

## 수강생이 자주 하는 실수

- "앱 만들어줘"처럼 범위가 너무 넓은 요청
- 기존 문서를 안 읽힌 상태에서 바로 구현 요청
- 한 번에 너무 큰 리팩토링 요청
- 검토 기준 없이 "더 좋게" 해달라고 요청

## 강의 팁

- Claude에게 먼저 "이 프로젝트에서 오늘 다룰 우선순위 3개만 정리해줘"라고 시키면 수강생의 불안을 낮출 수 있습니다.
- 구현 전에 반드시 문서 폴더를 보게 하면 결과 품질이 안정적입니다.
