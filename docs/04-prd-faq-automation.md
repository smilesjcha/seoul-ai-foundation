# PRD: 민원/FAQ 자동화 보조 서비스

## 1. 문제 정의

공공기관 또는 고객지원 조직에서는 반복적으로 유사한 민원과 FAQ 문의가 유입됩니다. 운영자는 같은 질문에 유사한 답변을 반복 작성하고, 문의 분류와 처리 우선순위 판단에도 많은 시간이 소요됩니다. 이로 인해 응답 속도와 품질 편차가 발생합니다.

본 서비스는 "운영자를 대체"하는 것이 아니라, 운영자가 더 빠르고 일관되게 응답할 수 있도록 돕는 AI 보조 시스템을 목표로 합니다.

## 2. 목표

- 반복 문의를 자동 분류한다.
- 기존 FAQ를 기반으로 적절한 답변 초안을 생성한다.
- 운영자가 검토 후 빠르게 확정할 수 있도록 지원한다.
- 응답 품질의 편차를 줄인다.

## 3. 비목표

- 완전 자동 발송 시스템 구축
- 개인정보가 포함된 실제 민원 데이터 처리
- 다국어 대응
- 기관별 정책 승인 시스템 완전 연동

## 4. 사용자

### 운영자

- 문의를 확인하고 적절한 응답을 작성해야 함
- 반복 업무 비중이 높음
- 응답 정확성과 톤을 모두 신경 써야 함

### 관리자

- 처리량과 응답 품질을 확인하고 싶어함
- 자주 발생하는 문의 주제를 파악하고 싶어함

### 시민/사용자

- 빠르고 이해하기 쉬운 답변을 원함

## 5. 핵심 기능

### 기능 1. 문의 목록 조회

- 최근 유입 문의를 목록으로 보여준다.
- 상태, 카테고리, 긴급도, 접수 시간을 확인할 수 있다.

### 기능 2. 문의 상세 확인

- 문의 원문, 채널, 작성자, 첨부 메모를 확인할 수 있다.
- Claude가 추론한 카테고리와 추천 FAQ를 함께 보여준다.

### 기능 3. 응답 초안 생성

- 선택된 문의에 대해 FAQ 기반 응답 초안을 생성한다.
- 응답에는 친절한 톤, 다음 액션, 주의 문구가 포함된다.

### 기능 4. 검토 포인트 제안

- 정책 확인 필요 여부
- 개인정보 포함 가능성
- 담당 부서 이관 필요 여부

## 6. MVP 범위

- Mock 데이터 기반 문의 목록/상세
- 카테고리 분류 로직
- FAQ 매칭
- 응답 초안 생성
- 운영자 화면에서 결과 확인

## 7. 사용자 흐름

1. 운영자가 문의 목록에 접속한다.
2. 새 문의를 선택한다.
3. 시스템이 카테고리, 추천 FAQ, 응답 초안을 제시한다.
4. 운영자가 내용을 수정하거나 승인한다.
5. 처리 상태를 `reviewed` 또는 `sent`로 변경한다.

## 8. 성공 지표

- 운영자 초안 작성 시간 50% 이상 단축
- 반복 문의 처리 시간 단축
- 응답 초안 재사용률 증가
- 운영자 만족도 향상

## 9. 리스크

- FAQ가 오래되어 잘못된 안내가 생성될 수 있음
- 민감한 문의에 기계적 응답이 생성될 수 있음
- 운영자 승인 없이 자동화 범위가 넓어질 경우 품질 사고 가능성 존재

## 10. 오픈 질문

- 실제 기관 정책 문구는 어디에서 관리할 것인가
- 응답 발송 전 승인 단계는 몇 단계가 필요한가
- 카테고리 체계는 기관별로 어떻게 커스터마이징할 것인가
